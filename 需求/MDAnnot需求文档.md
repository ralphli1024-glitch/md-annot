# MDAnnot — Obsidian Markdown 批注插件（需求文档）

## 背景与痛点

笔记已全面迁移到 Obsidian，但在阅读 `.md` 文件时存在以下问题：

- 现有的批注插件只支持 PDF，不支持 Markdown
- 阅读时需要在特定文字上划线/画圈/高亮/写批注来帮助记忆和关联

## 目标

开发一个 Obsidian 插件 **MDAnnot**，实现在 Markdown 文档上叠加批注层，批注锚定到文本内容，不破坏 `.md` 原文。

## 核心需求

### 编辑模式

选中文字后弹出浮动工具栏，支持三种操作：

| 操作 | 视觉表现 | 说明 |
|------|---------|------|
| **划线** | 文字下方波浪线 | 标记需要关注的词句 |
| **高亮** | 文字背景色高亮 | 突出关键信息 |
| **批注** | 文字高亮 + 右侧面板输入评论 | 定位文字并附加详细评论 |

三条规则：

- 批注信息不写入 `.md` 文件，存储在独立的 `.annotations/` 目录（`<文件名>.current.json` + 历史版本）
- 编辑模式下，被批注的文字通过 **CodeMirror 6 decorations** 显示标记：
  - 高亮 → 淡绿色背景
  - 划线 → 蓝色波浪线
  - 有批注 → 淡黄色背景 + 行号侧小圆点
- 可选设置：将高亮同步写入 `==高亮==` 语法、划线同步写入 `~~删除线~~` 语法，默认关闭

### 阅读模式

透明 Canvas 画布层覆盖在渲染的 Markdown 文本上方，支持：

- **编辑模式的批注可见**：用箭头指向被批注的文字位置
- 所有批注使用统一的颜色框区分
- 批注显示/隐藏开关
- 右侧批注面板在阅读模式下自动隐藏

### 锚定与丢失处理

批注基于 **文本内容指纹**（targetText + contextBefore + contextAfter 的哈希值）锚定，而非行号，确保编辑后不轻易错位。

匹配降级策略：

| 锚定状态 | 渲染颜色 | 说明 |
|---------|---------|------|
| `exact` | 蓝色/绿色 | 精确匹配，位置准确 |
| `fuzzy` | 橙色 | 原文被修改，已就近模糊匹配到相似文本 |
| `paragraph` | 橙色虚线框 | 行数变化，批注移至段落顶部 |
| `lost` | 红色虚线框 + 半透明 | 原文已被删除，固定在文档底部或侧面显示 |

所有丢失的批注保留历史记录，可查阅原始内容。

### 右侧批注面板

- 编辑模式下，文档存在批注时自动显示
- 可手动隐藏/显示（Ribbon 图标、命令面板 `Cmd+P`、快捷键 `Cmd+Shift+A`）
- 面板内容：所有批注的列表，每条显示类型/文字摘要/锚定状态/跳转按钮/删除按钮
- 支持筛选（全部/划线/高亮/批注）
- 支持导出当前文档批注为 Markdown（可选：导出全部文档批注）

### 设置面板包含

1. **显示控制**：阅读/编辑模式批注显示开关、面板自动显示开关
2. **批注样式**：默认高亮/划线/批注颜色、丢失批注颜色（固定红色）
4. **数据存储**：批注存储位置（库内 vs 插件目录）、历史版本保留策略、自动保存间隔
5. **Markdown 互操作**：是否同步写入 `==高亮==` / `~~划线~~` 语法到 `.md` 文件

## 技术方案

| 维度 | 选择 | 理由 |
|------|------|------|
| 语言 | TypeScript | Obsidian 插件唯一选择 |
| 构建 | esbuild | Obsidian 官方推荐 |
| 编辑器装饰 | `Plugin.registerEditorExtension()` | Obsidian 对 CM6 的封装 |
| 存储 | `app.vault.adapter` 读写 JSON 文件 | 透明、可随文档同步 |
| 外部依赖 | 无（零 API / 零数据库 / 零 SDK） | 完全离线运行 |

### 目录结构

```
.obsidian/plugins/md-annot/
├── main.js
├── manifest.json
└── styles.css
```

批注数据目录：

```
.annotations/
├── note.md.current.json
├── note.md.history/
│   ├── 20260713_1030.json
│   └── ...
└── index.json
```
