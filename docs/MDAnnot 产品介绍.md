# MDAnnot — Obsidian Markdown 批注插件

> 在 Markdown 上留下你的阅读痕迹——高亮、划线、评论，不改原文。

---

## 痛点与场景

### 痛点

笔记迁移到 Obsidian 后，阅读 `.md` 文件遇到了几个难以忍受的 gap：

- **无法在文字上直接划线标注**，想要高亮、划线、写评论帮助记忆，但 Obsidian 原生不支持
- **批注会破坏原文**，某些方案把批注信息塞进 Markdown 语法（`==高亮==`、`~~删除线~~`），污染原始文本，强迫阅读者接受批注
- 批注的重点内容无法导出

### 目标用户

| 用户 | 场景 |
|------|------|
| **知识工作者** | 阅读长文笔记时标注重点、记录思考 |
| **学生/研究人员** | 精读文献、做读书笔记、写评论 |
| **审阅者** | 在不修改原文的前提下，给文档添加标注和评论 |

### 适用场景

- 读书笔记：批注划线、写书评，不污染原文
- 技术文档评审：高亮疑问点，评论区讨论
- 会议纪要整理：标注待办事项关联的关键信息
- 学习笔记：多种颜色区分不同层级的知识重点
- 多设备阅读：Mac 上标注，手机上查看，批注自动同步

## 特点

| 特点 | 说明 |
|------|------|
| **不污染源文** | 批注数据独立存储在 `.obsidian/md_annot/` 目录，不修改 `.md` 原文。配合 Remotely Save 等同步插件开启配置文件夹同步，即可实现多设备批注自动同步 |
| **批注列表** | 右侧面板集中管理当前文档所有批注，支持按类型筛选、按位置排序 |
| **批注定位** | 点击任意批注卡片自动滚动到文档中对应文字位置，橙色呼吸闪烁动画提示 |
| **匹配度识别** | 每条批注显示锚定状态标签（精确/模糊/段落/丢失）和 10 格彩色置信度进度条 |
| **颜色配置** | 12 种预设色卡快速选色，高亮、划线、评论三种颜色独立配置 |
| **导出批注** | 支持导出当前文档批注和所有文档批注为 Markdown 格式 |
| **多设备同步** | 批注数据存储在 `.obsidian/md_annot/`，开启 Remotely Save「同步配置文件夹」即可跨设备自动同步批注 |

## 核心功能

### 编辑和阅读模式批注

编辑模式选中文字后弹出浮动工具栏，支持三种操作；阅读模式下选中文字同样弹出工具栏，渲染效果一致：

| 操作 | 视觉 | 说明 |
|------|------|------|
| **高亮** | 文字背景色标记（默认绿色） | 突出关键信息 |
| **划线** | 蓝色波浪下划线 | 标记需要关注的词句 |
| **评论** | 背景色 + 评论文本（默认黄色） | 定位文字并附加详细笔记 |

- 选中多行文字也能跨行创建批注
- 批注密集区域按钮自动高亮显示已有批注，重新选中时一键 toggle 覆盖/删除
- 超过 200 字符自动禁用按钮，避免误操作
- 批注数据不写入 `.md` 文件，存储在独立的 `.annotations/` 目录，随 vault 同步

### 文本锚定引擎

批注位置不依赖行号，而是基于**文本内容指纹**（FNV-1a 哈希 + 上下文）定位。编辑文档后分四级降级匹配：

```
精确匹配(EXACT) → 模糊匹配(FUZZY) → 段落匹配(PARAGRAPH) → 标记丢失(LOST)
```

**匹配流程**：

1. **EXACT 精确匹配** — 全文搜索 targetText，完整出现即算精确。多个匹配位时用选中文本前文（contextBefore）消歧。优先走 contentHash 快速路径（O(1) 校验）。
2. **FUZZY 模糊匹配** — 精确匹配失败（原文被修改），用 Levenshtein 编辑距离在文档中滑动窗口找最相似的文本段。相似度 > 0.7 判定匹配。
3. **PARAGRAPH 段落匹配** — 模糊也失败，取 targetText 中第一个有意义的长词，定位到该词所在段落（前后各一行）。
4. **LOST 标记丢失** — 前三步全失败，原文已被删除。保留原始数据供查阅。

**渲染效果**（编辑模式 CM6 装饰器 & 阅读模式 PostProcessor 一致）：

| 状态 | 视觉 |
|------|------|
| EXACT | 正常渲染（无额外修饰） |
| FUZZY | 类型底色不变 + 橙色虚线底纹 |
| PARAGRAPH | 行级橙色虚线框 |
| LOST | 半透明 + 删除线 + 红色虚线框 |

**匹配度识别**：侧面板每条批注卡片右侧显示 10 格彩色置信度进度条，颜色从绿→黄→橙→红渐变，扫一眼就能判断锚定状态：

```
精确：██████████ (全绿)
模糊：██████████ 前段绿→后段橙红
段落：██████████ 偏橙红
丢失：██████████ 前段黄→后段红
```

**性能优化**：编辑器内容变更时，CM6 ChangeTracker 捕获 delta 做 O(1) 增量位移，避免每次编辑都全文搜索。

### 批注列表面板

右侧面板集中管理当前文档所有批注：

- **类型筛选**：下拉框选择全部 / 高亮 / 划线 / 评论
- **排序**：按位置 / 高亮在前 / 评论在前
- **匹配度提示**：每项卡片显示锚定状态标签和 10 格彩色置信度进度条
- **点击定位**：点击任意批注卡片自动滚动到文档中对应文字位置并触发橙色呼吸闪烁动画
- **一键删除**：右上角垃圾桶按钮删除对应批注
- **导出批注**：底部「📥 导出批注」按钮，将当前文档批注导出为 `{文档名}_批注.md`

### Undo/Redo

跨平台快捷键（macOS: Cmd+Z, Windows: Ctrl+Z）：

- 撤销批注创建 / 批注删除
- Shift+Z 恢复撤销
- 50 步历史栈
- 编辑器文字编辑的原生 Undo 与批操作 Undo 互不干扰

### 设置面板

- **显示控制**：全局批注开关（关闭时清除所有渲染、隐藏图标和面板）、自动显示面板
- **颜色设置**：高亮 / 划线 / 评论三种颜色独立配置，3×4 色卡弹窗快速选色
- **颜色渲染模式**：全局统一颜色 vs 每个批注保留创建时的历史颜色
- **数据存储**：清空所有批注数据（到系统废纸篓）、清理失效缓存
- **导出所有批注**：遍历所有有批注的文档，按文档分组导出为 `{时间戳}_批注.md`，保存到仓库根目录

## 技术亮点

| 维度 | 方案 | 选择理由 |
|------|------|----------|
| **锚定** | FNV-1a 轻量哈希 + Levenshtein 编辑距离 | 前端实时计算无性能开销，O(1) 校验 |
| **匹配度** | 四级降级 + 10 格彩色进度条 | 视觉直观，一眼判断锚定状态 |
| **编辑器装饰** | CodeMirror 6 Decoration | 不影响原文，仅视觉层叠加 |
| **阅读模式** | MarkdownPostProcessor + DOM TreeWalker | 与编辑模式渲染效果一致 |
| **存储** | `.obsidian/md_annot/` 目录 JSON 文件 | 透明可同步，配合 Remotely Save 开启配置文件夹同步即可跨设备同步批注 |
| **外部依赖** | 无 | 完全离线运行，安装即用 |

### 架构概览

```
main.ts (入口/生命周期/事件编排)
  ├── annotation-manager.ts (CRUD/事件/UndoRedo)
  │   ├── data-models.ts (类型定义)
  │   ├── storage.ts (.annotations/ 读写)
  │   └── anchor-engine.ts (文本锚定)
  ├── floating-toolbar.ts (编辑/阅读模式选中工具栏)
  ├── reading-toolbar.ts (阅读模式工具栏，已废弃)
  ├── cm6-decorations.ts (编辑器装饰 + ChangeTracker 增量位移)
  ├── side-panel.ts (批注列表面板 / 筛选 / 排序 / 定位)
  └── settings-tab.ts (设置面板)
```
